# Sprint-1 As-Built Architecture (Agent Foundation)

This is the architecture **as implemented in Sprint 1**. Backup/restore/compression/encryption
**engines** are not built yet (Sprints 3–6); only the foundation that the rest of the product binds to
is in place. Each subsection links the authoritative design record (DR) and decision record (ADR).

## Components in this repository

| Component | Role |
|---|---|
| `beyz-backup-agent` | Long-running Windows service (Linux systemd prepared): config → logging → state → transport → enroll/heartbeat/task-poll → updater trigger. |
| `beyz-backup-updater` | **On-demand** binary (not a persistent service): verify → stage → swap → health-gate → rollback, driven by a crash-safe FSM. |
| Control-plane API (`api/openapi.yaml`) | The committed source of truth for the 6 agent endpoints; `pkg/proto` is generated from it. The SaaS backend is Sprint 2. |
| Shared kernel (`pkg/*`) | `proto`, `wireversion`, `manifest`, `hashing`, `storage` (interface only). |

## Control-plane API boundary

Six frozen `/v1/` endpoints — `enroll`, `register`, `heartbeat`, `tasks`, `tasks/{id}/ack`,
`tasks/{id}/status` — defined in `api/openapi.yaml` (OpenAPI 3.1, the **single source of truth**).
The Go client is generated by `oapi-codegen` with a CI drift gate, and validated against a Prism mock
(`task contract`). Sprint-1 exercises the enroll → register → heartbeat → poll happy path; `ack`/
`status` are wired and tested against a noop task. See **[DR-05](design/DR-05-protocol-versioning.md)**.

## Protocol versioning & compatibility

Every request carries `X-Agent-Version` (ldflags-injected from the repo-root `VERSION`) and
`X-Protocol-Version`. An out-of-window agent receives **`426 Upgrade Required`** (terminal, routed to
the updater) rather than looping. Envelopes are forward-compatible (`additionalProperties`, integer
`schema_version`s, reserved nullable fields) so later sprints extend the wire without breaking deployed
agents. Compatibility window policy = N − 3 minor versions. See
**[DR-05](design/DR-05-protocol-versioning.md)** · [ADR-004](ADR/ADR-004-Protocol-Versioning.md).

## Identity & device binding

Three independent values bound at enrollment: a server-issued opaque **`device_id`** (primary), an
agent-generated write-once **`device_guid`** (UUIDv4 continuity anchor that survives reinstall/DB
rebuild), and a **`spki_sha256`** credential fingerprint over a local **ECDSA P-256** key + CSR.
Advisory hardware signals are hashed (`sha256:<hex>`) for clone detection only — never raw, never the
license key. See **[DR-02](design/DR-02-enrollment-and-identity.md)** ·
[ADR-003](ADR/ADR-003-Agent-Identity-Model.md).

## Enrollment & authentication

A **single-use enrollment token** (from `BEYZ_ENROLLMENT_TOKEN` or the installer's one-shot
`state/enroll-token` file — never `config.yaml`, never logged) is exchanged at `POST /v1/enroll` for a
`device_id`, a tenant-bound certificate, and a bearer `agent_session_token`. `401`/`409` are terminal
token rejections; `426` routes to the updater. The token file is deleted on a definitive outcome and
purged on an already-enrolled device. See **[DR-02](design/DR-02-enrollment-and-identity.md)**.

## Agent state store & secret protection

A single bbolt file (`state/agent-state.db`) plus a write-once `device.guid` sidecar. Secret values
(private key, session token, license) are **wrapped** (Windows DPAPI machine-scope; fail-closed on
non-Windows) and non-secret values are plaintext — both behind an **ACL-locked folder** that is the
real boundary (`SYSTEM` + `Administrators` only on `state\`). The key-classification API prevents
writing a secret as plaintext. See **[DR-01](design/DR-01-key-management.md)** ·
[ADR-001](ADR/ADR-001-Encryption-Recovery-Model.md).

## Update trust model

The updater verifies a **real, enforcing Ed25519 signature** over the RFC 8785 (JCS) canonical
manifest against a **compile-time embedded public-key *set*** (selected by `key_id`, with embedded +
manifest revocation), then verifies the binary's **BLAKE3 + SHA-256** hashes taken only from the
signed manifest, enforces **anti-rollback** + a `min_supported_version` floor, performs an **atomic
`MoveFileEx` swap**, gates on a **90s health check**, and **rolls back** to an integrity-checked backup
on failure — all driven by a crash-safe persisted FSM. Manifest fetch is SPKI-pinned; the binary may
come from any HTTPS host (integrity is from the signature, not transport). See
**[DR-03](design/DR-03-update-trust-and-rotation.md)** ·
[ADR-002](ADR/ADR-002-Update-Signing-Trust-Model.md).

## Multi-tenant isolation

`tenant_id` (+ `parent_org_id` for MSP, `region` for residency) is **server-authoritative and bound
into the agent certificate**; `location_id` is advisory and **never** cert-bound. The agent produces
and persists the binding; server-side enforcement (shared-schema + `tenant_id`-leading keys + RLS) is
Sprint 2. See **[DR-04](design/DR-04-tenancy-and-isolation.md)** ·
[ADR-006](ADR/ADR-006-Location-Site-Scoping-and-RBAC-Boundary.md).

## Audit trail

Security events are written to a `security.log` stream and a bbolt audit-spool with a frozen schema and
a **per-record BLAKE3 hash chain** anchored to a device-bound genesis (anti-graft), with a controlled
event vocabulary enforced at emit and verify time. Tamper-**evident** in Sprint 1; off-device WORM
anchoring is Sprint 8. See **[DR-06](design/DR-06-audit-event-schema.md)**.

## Installer & on-disk layout (Windows)

Inno Setup installs both binaries to `C:\Program Files\BeyzBackup\`, builds an **ACL-hardened**
`C:\ProgramData\BeyzBackup\{state,logs,update}` tree (fail-closed; inheritance broken; locale-independent
SIDs), registers and starts the **single `BeyzBackupAgent`** service (LocalSystem, auto-start +
recovery), and installs the updater **as a binary, not a service**. The enrollment token is written via
a raw file write and consumed on first run; artifacts are unsigned but **signing-ready** (Authenticode
is separate from the Ed25519 update root). Full detail: **[INSTALL.md](INSTALL.md)** ·
`docs/sprint-1/07-INSTALLER.md`.

## CI security gates

Blocking, non-bypassable merge checks: `lint`, `test` + coverage gate, `race`, `security`
(`task test:negative`), `contract`, `drift`, `gitleaks`, `govulncheck`, `cross`. Windows/Pester jobs
run but are non-blocking until stabilized. Coverage gate: ≥70% overall, ≥85% on the security-critical
packages. See [SECURITY.md](SECURITY.md) and `docs/sprint-1/{06-CI.md,09-SECURITY-GATES.md}`.
