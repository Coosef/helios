# S1-T31 — CI Pipeline

The CI lives in `.github/workflows/ci.yml`, triggered on `pull_request` and `push`
to `main`, with `concurrency: cancel-in-progress`. All CI-critical tooling is **pinned**
(no `@latest`). Jobs invoke the **Taskfile** where a task is the source of truth
(`contract`, `drift`), so **local == CI**.

## govulncheck triage (runbook)

`govulncheck ./...` is a required gate. A new upstream CVE can turn it red on an
unrelated PR. When that happens:

1. Read the report — govulncheck only flags CVEs **reachable in our call graph**, so a
   hit is usually actionable.
2. **In-graph & fixable:** bump the dependency (`go get <mod>@<fixed>` + `go mod tidy`),
   re-run, open the dep-bump in the same or a follow-up PR.
3. **Not reachable / no fix yet:** record a time-boxed waiver in `docs/sprint-1/06-CI.md`
   (this section) with the CVE id, the affected module, why it is not reachable, and a
   review date; only then add a scoped suppression. Do **not** silence the whole job.

## Forward item — FI-T31-2: Windows test portability (CI-discovered)

The `windows-test` job is wired **non-blocking** (`continue-on-error: true`) for now.
Five tests use `os.Chmod(dir, 0o500)` to force a write failure
(`internal/health`, `internal/updater/app` ×2, `internal/updater/swap`,
`internal/agent/app`). Windows does **not** enforce Unix directory permissions, so the
forced-failure assertions would fail there — and one (`internal/updater/swap/errors_test.go`)
has no `cannot chmod` skip at all. These are **test-portability** issues, not production
bugs. The fix (a `runtime.GOOS == "windows"` skip on those Unix-permission tests) is a
**separate reviewed task**; once `windows-test` is green it becomes a **required** gate.
The job already RUNS on every CI, giving the first real Windows execution of the
`MoveFileEx` swap and `service.Status` paths.
